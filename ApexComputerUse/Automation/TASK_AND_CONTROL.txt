# ApexComputerUse: Complete Usage & Control Guide

## **Starting ApexComputerUse**

### **Desktop Application**
```bash
dotnet run --project ApexComputerUse
```

Opens a WinForms application with tabs:
- **Remote Control** — Start HTTP/Named Pipe servers, display API key
- **Find & Execute** — manual UI for element discovery and actions
- **Console** — logs and command output
- **Model** — Download models, load vision/audio LLMs

### **Configuration**

**`appsettings.json`** (next to executable):
```json
{
  "HttpPort": 8080,                    // HTTP server listen port
  "HttpBindAll": false,                // false: localhost only | true: all interfaces
  "PipeName": "ApexComputerUse",      // Named pipe name
  "LogLevel": "Information",           // Debug, Information, Warning, Error
  "EnableShellRun": true,              // Enable POST /run shell execution
  "TelegramToken": "",                 // Telegram bot token
  "TestRunnerExePath": "..."           // Path to TestRunner executable
}
```

**Environment Variables** (override `appsettings.json`):
```bash
APEX_HTTP_PORT=8080
APEX_HTTP_BIND_ALL=false
APEX_PIPE_NAME=ApexComputerUse
APEX_LOG_LEVEL=Information
APEX_ENABLE_SHELL_RUN=true
APEX_API_KEY=<auto-generated-or-override>
APEX_ALLOWED_CHAT_IDS=<comma-separated>
APEX_TELEGRAM_TOKEN=<token>
APEX_MODEL_PATH=C:\models\vision.gguf
APEX_MMPROJ_PATH=C:\models\mmproj.gguf
```

**User Preferences** (`%APPDATA%\ApexComputerUse/`):
```
settings.json          — GUI state, ModelPath, MmProjPath, ApiKey, AllowedChatIds
ai-settings.json       — AI provider API keys (OpenAI, Anthropic, DeepSeek, Grok, etc.)
logs/apex-YYYYMMDD.log — Daily rotating logs (7-day retention)
```

## **Authentication**

Every HTTP request requires an API key. Three equivalent methods:

```bash
# Authorization header (recommended)
curl -H "Authorization: Bearer <key>" http://localhost:8080/ping

# X-Api-Key header
curl -H "X-Api-Key: <key>" http://localhost:8080/ping

# Query parameter (browser links / quick tests only)
curl "http://localhost:8080/ping?apiKey=<key>"
```

- Generated automatically on first launch
- Display in the **Remote Control** tab
- Disable by clearing the API Key field (local development only)

### **Element Discovery**

```bash
# Get element tree (nested JSON with IDs)
curl -H "X-Api-Key: <key>" http://localhost:8080/elements

# Onscreen only (80% fewer elements on browsers)
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?onscreen=true"

# Shallow overview (depth=2)
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?depth=2&onscreen=true"

# Expand specific node (preserves map, IDs stable)
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?id=708379645&depth=2"

# Filter by ControlType
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?type=Button"

# Text search across Name, AutomationId, Value
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?match=submit&onscreen=true&depth=1"

# Collapse wrapper chains (remove identity-less Pane/Group/Custom single-child nodes)
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?collapseChains=true"

# Add ancestor breadcrumbs
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?includePath=true"

# Include Value pattern + HelpText on nodes
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?properties=extra"

# Combine all
curl -H "X-Api-Key: <key>" "http://localhost:8080/elements?onscreen=true&collapseChains=true&type=Button&depth=2&match=save&properties=extra"
```

### **Window & Element Finding**

```bash

     -H "X-Api-Key: <key>" \

     -H "X-Api-Key: <key>" \
     -H "Content-Type: application/json" \

     -H "X-Api-Key: <key>" \
     -H "Content-Type: application/json" \

```

### **Serve Mode** (HTTP control surface)
```bash
TestRunner --serve --serve-port 8765 --serve-host 127.0.0.1
```

External callers POST to:
- `POST /tests` — list or run all tests
- `POST /tests/{testId}` — run single test by ID
- `POST /bridge/restart` — restart ApexComputerUse process
- `GET /results` — history of recent test runs
- `POST /shutdown` — graceful shutdown

---

### **Key Metrics Tracked**

1. **Per-test latency** (`ElapsedMs`) — detects regressions
2. **Pass/fail count** — cycle stability
3. **Build time** — development iteration speed
4. **Bridge API readiness** — server startup overhead
5. **Suite execution time** — overall throughput

**Default benchmark targets:** 25 cycles, report every 10 cycles, skip previously-passed tests in re-runs.
